HTML 页面结构元素 = 把“整张网页”拆成**语义化、可访问、易维护**的积木块。  
HTML5 新增了一大批“结构标签”，目的就是让你不再清一色用 `<div id="header">` 这种“无语义”写法。

---

### 1. 一页完整的“骨架”
```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="utf-8">
  <title>语义化示例</title>
</head>
<body>
  <header>…</header>
  <nav>…</nav>
  <main>
    <article>
      <header>…</header>
      <section>…</section>
      <section>…</section>
    </article>
    <aside>…</aside>
  </main>
  <footer>…</footer>
</body>
</html>
```

---

### 2. 结构元素速查表

| 标签        | 语义                   | 典型放置位置                 | 备注/注意                               |
| ----------- | ---------------------- | ---------------------------- | --------------------------------------- |
| `<header>`  | 头部**区域**           | 页头 / 文章标题栏            | 一页面可多个（article、section 也能用） |
| `<nav>`     | **主导航**链接组       | 页头横向菜单、侧边目录       | 不是“所有链接”，而是“主要导航区块”      |
| `<main>`    | **主体唯一核心**       | 页面唯一！                   | 一个 HTML 文档只能出现一个 `<main>`     |
| `<section>` | **按主题分节**         | 章节、选项卡面板             | 必须有标题（h1-h6）才语义成立           |
| `<article>` | **可独立分发**的内容块 | 博客正文、论坛帖子、商品卡片 | 可嵌套 section，也可被 section 嵌套     |
| `<aside>`   | **旁支/补充**信息      | 侧边栏、广告、引用、相关推荐 | 与周围内容“弱相关”                      |
| `<footer>`  | **尾部**区域           | 页脚 / 文章底部作者/版权     | 同 header，可存在多个                   |
| `<div>`     | **无语义容器**         | 仅当以上标签都不合适才回退   | 配合 class 做样式钩子                   |
| `<h1>-<h6>` | **标题级别**           | 各区块的标题                 | 顺序别跳级，一个独立内容块允许 h1       |

---

### 3. 细节与最佳实践

1. **标题先行**  
   section / article 建议**第一行就是 `<h*>`**，否则屏幕阅读器会报“无标题区域”。

2. **main 只能有一个**  
   搜索引擎/辅助技术用它快速定位“跳过导航”入口。

3. **nav 别滥用**  
   页脚的小链接（版权、隐私）通常放 `<footer>` 即可，不必再套 `<nav>`。

4. **article vs section 谁嵌套谁？**  
   - 能脱离上下文仍完整 → `article`  
   - 只是按逻辑分节 → `section`  
   例：  
   ```html
   <article><!-- 一整篇评测 -->
     <h1>iPhone 15 评测</h1>
     <section><!-- 分节 -->
       <h2>外观设计</h2>
     </section>
     <section>
       <h2>性能跑分</h2>
     </section>
   </article>
   ```

5. **aside 位置灵活**  
   放在 article 内部 = 与该文章相关（如作者注）；  
   放在 body 层级 = 全站侧边栏。

6. **旧版 IE 不认新标签？**  
   引入 html5shiv / reset css 即可：
   ```html
   <!--[if lt IE 9]>
     <script src="https://cdn.jsdelivr.net/npm/html5shiv@3.7.3/dist/html5shiv.min.js"></script>
   <![endif]-->
   ```

### 4. 一张图记心里（纯文字版）
```
body
├─ header
│  ├─ h1 站点标题
│  └─ nav  全局导航
├─ main
│  ├─ article
│  │  ├─ header 文章标题
│  │  ├─ section 正文段落1
│  │  └─ section 正文段落2
│  └─ aside 相关文章推荐
└─ footer 版权备案
```

---

### 5. 总结一句话
先写**语义结构标签**（header/nav/main/section/article/aside/footer），  
再用**div 做样式补充**，
最后加**标题 h1-h6** 给每块起名字——  
这样搜索引擎、屏幕阅读器、未来的你，都能 3 秒看懂页面骨架。
