在 HTML 中，`<ul>`、`<ol>` 和 `<li>` 是用于创建**列表**的标签，它们分别代表：

---

### ✅ `<ul>`：无序列表（Unordered List）
- 默认显示为**圆点符号**（•）
- 用于项目顺序不重要的列表

```html
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橙子</li>
</ul>
```

**效果：**
- 苹果
- 香蕉
- 橙子

---

### ✅ `<ol>`：有序列表（Ordered List）
- 默认显示为**数字编号**（1, 2, 3...）
- 用于项目顺序重要的列表

```html
<ol>
  <li>第一步</li>
  <li>第二步</li>
  <li>第三步</li>
</ol>
```

**效果：**
1. 第一步  
2. 第二步  
3. 第三步

---

### ✅ `<li>`：列表项（List Item）
- 必须嵌套在 `<ul>` 或 `<ol>` 中使用
- 每个 `<li>` 是一个列表项

---

### ✅ 嵌套列表示例（组合使用）

```html
<ul>
  <li>水果
    <ul>
      <li>苹果</li>
      <li>香蕉</li>
    </ul>
  </li>
  <li>蔬菜
    <ol>
      <li>胡萝卜</li>
      <li>西兰花</li>
    </ol>
  </li>
</ul>
```

---

### ✅ 可选属性（简要）
- `<ol type="A">`：用大写字母编号（A, B, C...）
- `<ol start="5">`：从第5项开始编号
- `<ul style="list-style-type: square;">`：方形符号

---

在 HTML 中，`<dl>`、`<dt>` 和 `<dd>` 是用于创建**描述列表（description list）**的标签，适合展示“术语-定义”或“键-值”对结构。

---

### ✅ 标签含义

| 标签   | 全称                | 作用                 |
| ------ | ------------------- | -------------------- |
| `<dl>` | Description List    | 描述列表的容器       |
| `<dt>` | Description Term    | 被描述的“术语”或“键” |
| `<dd>` | Description Details | 对术语的“定义”或“值” |

---

### ✅ 基本用法示例

```html
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言，用于创建网页结构。</dd>

  <dt>CSS</dt>
  <dd>层叠样式表，用于控制网页的外观和布局。</dd>

  <dt>JavaScript</dt>
  <dd>一种脚本语言，用于实现网页的交互功能。</dd>
</dl>
```

**浏览器默认渲染效果（带缩进）：**

HTML  
  超文本标记语言，用于创建网页结构。  
CSS  
  层叠样式表，用于控制网页的外观和布局。  
JavaScript  
  一种脚本语言，用于实现网页的交互功能。

---

### ✅ 一个术语多个定义

```html
<dl>
  <dt>Java</dt>
  <dd>一种编程语言。</dd>
  <dd>一种咖啡豆。</dd>
</dl>
```

---

### ✅ 多个术语一个定义

```html
<dl>
  <dt>HTTP</dt>
  <dt>HTTPS</dt>
  <dd>用于传输网页数据的协议，HTTPS 是加密的 HTTP。</dd>
</dl>
```

---

### ✅ 可以嵌套其他标签

```html
<dl>
  <dt><code>console.log()</code></dt>
  <dd>
    <p>向浏览器控制台输出信息。</p>
    <pre>console.log("Hello, world!");</pre>
  </dd>
</dl>
```

---

### ✅ 总结一句话
- `<dl>` 是**描述列表容器**
- `<dt>` 是**术语/键**
- `<dd>` 是**定义/值**

适合用来做**词汇表、FAQ、配置说明、元信息展示**等。

------------------------------------------------
1. 语义一句话记住
------------------------------------------------
- `<dl>`  Definition List  “列表容器”  
- `<dt>`  Definition Term  “被解释的词 / 问题 / 名称”  
- `<dd>`  Definition Description “解释 / 答案 / 值”  

HTML Living Standard 原文：  
「The dl element represents an association list consisting of zero or more name-value groups。」  
—— 不再局限于“字典定义”，任何“名-值”或“问题-答案”关联都可使用。

------------------------------------------------
2. 语法细则（WHATWG & W3C 双标准一致）
------------------------------------------------
- `<dl>` 只能包含 **零个或多个** `<dt>` / `<dd>` 标签，**不再允许直接放文本或 div**。  
- 一个 “名-值组” 允许：  
  a) 1 个 `<dt>` + 1 个 `<dd>`  
  b) n 个 `<dt>` 共享 1 个 `<dd>`（同义词、缩写）  
  c) 1 个 `<dt>` + n 个 `<dd>`（多段解释）  
- `<dt>` 与 `<dd>` 之间 **不允许出现“裸文本”**，但允许嵌套 **流内容**（flow content），例如 `<p>`、`<ul>`、`<article>`、`<header>` 等。  
- 标签闭合：HTML5 可省闭合标签，但 **建议显式闭合**，避免白边与 CSS 意外。

------------------------------------------------
3. 默认渲染 & CSS 清零
------------------------------------------------
浏览器 UA 样式（以 Chrome 为例）：

```css
dl { margin-block: 1em; }
dt { font-weight: bold; }
dd { margin-inline-start: 40px; }   /* 写死 40px，非 em */
```

清零写法（现代方案）：

```css
dl, dt, dd { margin: 0; padding: 0; }
dt { font-weight: unset; }
```

------------------------------------------------
4. 结构化用法速查表
------------------------------------------------
4.1 最简定义  
```html
<dl>
  <dt>HTTP</dt>
  <dd>HyperText Transfer Protocol</dd>
</dl>
```

4.2 多词同义  
```html
<dl>
  <dt>URL</dt>
  <dt>Uniform Resource Locator</dt>
  <dd>因特网上标准的资源地址格式。</dd>
</dl>
```

4.3 多段解释  
```html
<dl>
  <dt>Cookie</dt>
  <dd>小型文本文件，保存在用户本地终端。</dd>
  <dd>可用于会话管理、个性化设置、追踪分析。</dd>
</dl>
```

4.4 嵌套列表（合法且常用）  
```html
<dl>
  <dt>前端技术栈</dt>
  <dd>
    <ul>
      <li>HTML：结构</li>
      <li>CSS：表现</li>
      <li>JavaScript：行为</li>
    </ul>
  </dd>
</dl>
```

4.5 分组 + 微数据（搜索引擎富结果）  
```html
<dl itemscope itemtype="https://schema.org/FAQPage">
  <dt itemprop="name">退货政策</dt>
  <dd itemprop="acceptedAnswer" itemscope
      itemtype="https://schema.org/Answer">
    <p itemprop="text">收货后 7 天内可无理由退货。</p>
  </dd>
</dl>
```

4.6 脚本式对话（编剧场景）  
```html
<dl class="script">
  <dt>Alice</dt>
  <dd>你昨天看比赛了吗？</dd>

  <dt>Bob</dt>
  <dd>看了，绝杀太燃！</dd>
</dl>
```

5. 可访问性与 SEO
------------------------------------------------
- 屏幕阅读器（NVDA/VoiceOver）会在 `<dl>` 开头朗读 “description list with X items”，比 `<div>` 裸排更易导航。  
- 避免仅用 `<dt>` 做标题、`<dd>` 做段落 —— 如果顺序关系不强，优先考虑 `<section>` + `<h*>`.  
- 关键词尽量放在 `<dt>` 中，搜索引擎会把 `<dt>` 视为“术语”，`<dd>` 视为“释义”。  
- 使用 `id` + `aria-describedby` 可建立外部链接与定义的关联：

```html
<p>我们使用 <a href="#def-http" aria-describedby="def-http">HTTP</a> 传输网页。</p>
…
<dl>
  <dt id="def-http">HTTP</dt>
  <dd id="def-http-desc">超文本传输协议……</dd>
</dl>
```

------------------------------------------------
6. 与 ARIA 结合（可选增强）
------------------------------------------------
- 不要给 `<dl>` 加 `role="list"`，浏览器已隐式映射 `role="descriptionlist"`。  
- 如果拆成 `<div class="dt">` 做布局，必须手动补回：

```html
<div role="descriptionlist">
  <div role="term">Name</div>
  <div role="definition">Value</div>
</div>
```

------------------------------------------------
7. 常见坑与调试
------------------------------------------------
1. 旧教材写 `<dl compact>` 已废弃，无法用 CSS 还原，请用 `display: grid` 自行压紧。  
2. 想在 `<dt>` 前加自动编号：  
   ```css
   dl { counter-reset: term; }
   dt::before { counter-increment: term; content: counter(term) ". "; }
   ```
3.  flex/grid 布局时，忘记给 `<dd>` 设置 `margin: 0` 会出现意外缩进。  
4.  移动端希望 `<dt>` 与 `<dd>` 左右并排：  
   ```css
   dl.def-inline { display: grid; grid-template-columns: auto 1fr; gap: .5em 1em; }
   dt { grid-column: 1; }
   dd { grid-column: 2; }
   ```

------------------------------------------------
8. 一句话总结
------------------------------------------------
`<dl>`、`<dt>`、`<dd>` 不是“冷门标签”，而是 HTML 原生提供的 **语义化键值对容器**。  
任何“名-值”“问题-答案”“元数据-值”场景，优先使用它们，再辅以 CSS/ARIA/微数据，就能同时获得 **简洁代码、良好 SEO、无障碍支持** 三重收益。
