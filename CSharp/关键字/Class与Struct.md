一句话口诀  
“**小、短、单、16、不装箱**——五样全齐才配做 `struct`，其余通通写 `class`。”

| 维度     | class（引用类型）             | struct（值类型）                             |
| -------- | ----------------------------- | -------------------------------------------- |
| 分配位置 | 总是堆上，GC 回收             | 栈或内联，随作用域弹走                       |
| 数组开销 | 元素只是引用，额外指针+堆对象 | 元素内联，一块连续内存，局部性更好           |
| 类型转换 | cast 无装箱                   | 到接口/基对象就装箱，GC 压力                 |
| 复制成本 | 只拷引用（8 B）               | 拷全部字段，越大越贵                         |
| 参数传递 | 按引用                        | 按值（隐式整份拷贝）                         |
| 易变性   | 可变是常态                    | 可变会让隐式副本“掉”数据，**应设计成不可变** |

微软给的硬杠杠  
❌ 别写 struct，除非同时满足：  
1. 逻辑上是“单个值”（像 int、Point）  
2. 实例 ≤ 16 B  
3. 不可变  
4. 不会频繁被装箱  
5. 寿命短或常嵌在其他对象里

不满足？——老老实实 `class`/`record class`。
